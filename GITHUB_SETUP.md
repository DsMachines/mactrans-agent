# GitHub Setup Documentation

This document captures the minimal Git workflow we used for the **mactrans-agent** project. Keep it handy for future contributions.

## Quick Reference

```bash
# 1️⃣ Stage all changes
git add .

# 2️⃣ Commit with a concise message
git commit -m "<your message>"

# 3️⃣ Ensure the remote is set (run once)
# If the remote already exists, you can skip this step.
# Replace the URL if your fork/clone differs.
 git remote add origin https://github.com/DsMachines/mactrans-agent.git

# 4️⃣ Use the main branch (rename if needed)
 git branch -M main

# 5️⃣ Push to GitHub and set upstream
 git push -u origin main
```

## Notes
- The repository already tracks `origin` pointing to `https://github.com/DsMachines/mactrans-agent.git`.
- The `main` branch is configured to track `origin/main` after the first push.
- Git may warn about line‑ending conversion (LF → CRLF) on Windows; this is normal.
- Adjust the remote URL if you fork the repo or use a different remote name.

## Checklist for New Contributions
1. Pull the latest changes: `git pull`.
2. Create a new branch for your feature: `git checkout -b my-feature`.
3. Follow the **Quick Reference** steps above.
4. Open a Pull Request on GitHub after pushing.
