# GitHub Workflow Guide

This document captures the minimal Git commands we used for the **mactrans-agent** project. Keep it handy for any future updates.

## 1️⃣ Initial Setup (already done)
```bash
# Remote URL (already added)
git remote add origin https://github.com/DsMachines/mactrans-agent.git
# Ensure we are on the main branch
git branch -M main
```

## 2️⃣ Making Changes
1. **Stage all modified files**
```bash
git add .
```
2. **Commit with a descriptive message**
```bash
git commit -m "<your message>"
```
   - Example: `git commit -m "Setup phase completed"`

## 3️⃣ Push to GitHub
```bash
git push -u origin main
```
- The `-u` flag sets the upstream so future `git push`/`git pull` can be run without specifying the remote/branch.

## 4️⃣ Common Gotchas (Windows)
- Git may warn about line‑ending conversion (LF → CRLF). This is normal; you can ignore the warning or configure `core.autocrlf` if needed.
- Ensure the **.env.local** file is listed in `.gitignore` to avoid committing secrets.

## 5️⃣ Quick Checklist for Future Commits
- [ ] Pull the latest `main` (`git pull`).
- [ ] Create a feature branch if working on a bigger change.
- [ ] Run `git status` to verify staged files.
- [ ] Use clear, concise commit messages.
- [ ] Push and verify on GitHub.
